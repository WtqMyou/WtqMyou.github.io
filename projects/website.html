VitePress 网站日常维护备忘录 ​

启动本地预览 ​

  1. 打开终端（Git Bash）。
  2. 进入项目文件夹：
    bash
    cd ~/my_wiki
  3. 启动本地服务：
    bash
    npm run docs:dev
  4. 打开浏览器，访问 http://localhost:5173。接下来任何修改，保存后都会实时刷新。

修改网站内容 ​

1. 修改现有文章 / 主页文字 ​

  • 改主页：用记事本或 VS Code 打开 my_wiki/index.md，修改两条 --- 之间的文字（这是主页卡片和标题配置）。
  • 改笔记：直接打开 my_wiki/notes/ 里面的 .md 文件（比如 fb.md），正常修改里面的文本就行。

2. 添加全新的文章 ​

假设你想在“学习笔记”里加一篇《微积分》：

  1. 在 my_wiki/notes/ 文件夹下新建一个文件，命名为 calculus.md。
  2. 打开它，写上你的标题和内容（比如 # 微积分复习）。
  3. 关键一步（更新左侧目录树）：打开 my_wiki/.vitepress/config.mts，找到 sidebar 里面的 '/notes/': [...]，加上这篇新文章的入口：
    typescript
    items: [
  { text: '复变函数与积分变换', link: '/notes/fb' },
  { text: '微积分', link: '/notes/calculus' } // <-- 加上这一行
]

3. 添加新的 PDF 或图片 ​

  • 任何图片、PDF 文件，必须直接扔进 my_wiki/public/ 文件夹里。
  • 在文章中引用时，路径前面只加一个 /，不要写 public。
  • 例如，放入了 new-note.pdf，在文章里就这么写：
    html
    <iframe src="/new-note.pdf" style="width: 100%; height: 600px; border: none;"></iframe>

第三阶段：一键发布到互联网 ​

  1. 停止本地预览：在终端里按 Ctrl + C，然后按 Y 回车确认。
  2. 生成最新的网页文件：
    bash
    npm run docs:build
  3. 进入生成的打包文件夹：
    bash
    cd .vitepress/dist
  4. 执行强推四连（直接复制运行）：
    bash
    git init
git add -A
git commit -m "update website"
git push -f git@github.com:WtqMyou/WtqMyou.github.io.git master:main
  5. 退回到根目录：
    bash
    cd ../..
